## Prerequisites

Before installing bdg, ensure you have the following:

<CardGroup cols={2}>
  <Card title="Node.js" icon="node-js">
    Version 20.0.0 or higher required
  </Card>
  <Card title="npm/yarn/pnpm" icon="box">
    Package manager of your choice
  </Card>
  <Card title="Chrome" icon="chrome">
    Google Chrome or Chromium installed
  </Card>
  <Card title="Platform" icon="desktop">
    macOS, Linux, or Windows (WSL only)
  </Card>
</CardGroup>

<Note>
  **Node.js requirement:** bdg requires Node.js 20.0.0 or higher. Check your version with `node --version`.
</Note>

## Install bdg

Choose your preferred package manager:

<CodeGroup>

```bash npm
npm install -g browser-debugger-cli@alpha
```

```bash yarn
yarn global add browser-debugger-cli@alpha
```

```bash pnpm
pnpm add -g browser-debugger-cli@alpha
```

</CodeGroup>

<Warning>
  Currently installing from the `@alpha` tag. This will change to stable once v1.0 is released.
</Warning>

## Verify Installation

Confirm bdg is installed correctly:

```bash
bdg --help
```

You should see the command reference with available commands and options.

Check the installed version:

```bash
bdg --version
```

## Platform Support

### Supported Platforms

<AccordionGroup>
  <Accordion title="macOS" icon="apple">
    ✅ **Fully supported**
    
    bdg works out of the box on macOS. Chrome will be automatically located in standard installation paths.
  </Accordion>
  
  <Accordion title="Linux" icon="linux">
    ✅ **Fully supported**
    
    Works on all major distributions. Ensure Chrome or Chromium is installed:
    
    ```bash
    # Debian/Ubuntu
    sudo apt install chromium-browser
    
    # Fedora/RHEL
    sudo dnf install chromium
    
    # Arch
    sudo pacman -S chromium
    ```
  </Accordion>
  
  <Accordion title="Windows (WSL)" icon="windows">
    ✅ **Supported via WSL**
    
    Run bdg in Windows Subsystem for Linux (WSL 2 recommended):
    
    1. Install WSL: Follow [Microsoft's WSL installation guide](https://learn.microsoft.com/en-us/windows/wsl/install)
    2. Install Node.js in WSL
    3. Install Chrome for Linux in WSL
    4. Install bdg using npm/yarn/pnpm
  </Accordion>
  
  <Accordion title="Windows (Native)" icon="windows">
    ❌ **Not yet supported**
    
    PowerShell and Git Bash are not currently supported. Use WSL as a workaround.
  </Accordion>
</AccordionGroup>

## Troubleshooting

### Command not found

If you get `command not found: bdg` after installation:

<Steps>
  <Step title="Check npm global path">
    Ensure npm's global bin directory is in your PATH:
    
    ```bash
    npm config get prefix
    ```
    
    Add `<prefix>/bin` to your PATH if missing.
  </Step>
  
  <Step title="Verify installation">
    Check if the package is installed globally:
    
    ```bash
    npm list -g browser-debugger-cli
    ```
  </Step>
  
  <Step title="Reinstall if needed">
    Try reinstalling with verbose output:
    
    ```bash
    npm install -g browser-debugger-cli@alpha --verbose
    ```
  </Step>
</Steps>

### Node.js version mismatch

If you see an error about Node.js version:

```text
Error: The engine "node" is incompatible with this module.
```

**Solution:** Upgrade Node.js to version 20.0.0 or higher:

<CodeGroup>

```bash nvm
# Using nvm (recommended)
nvm install 20
nvm use 20
```

```bash direct
# Direct download
# Visit https://nodejs.org/ and download Node.js 20 LTS
```

</CodeGroup>

### Chrome not found

If bdg can't locate Chrome:

```text
Error: Chrome not found
```

<AccordionGroup>
  <Accordion title="macOS">
    Ensure Chrome is installed in the standard location:
    
    ```bash
    ls -la /Applications/Google\ Chrome.app
    ```
    
    If Chrome is elsewhere, bdg will attempt to locate it automatically via `chrome-launcher`.
  </Accordion>
  
  <Accordion title="Linux">
    Install Chrome or Chromium:
    
    ```bash
    # Download Chrome
    wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
    sudo dpkg -i google-chrome-stable_current_amd64.deb
    
    # Or install Chromium
    sudo apt install chromium-browser
    ```
  </Accordion>
  
  <Accordion title="WSL">
    Install Chrome for Linux inside WSL, not Windows Chrome:
    
    ```bash
    wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
    sudo dpkg -i google-chrome-stable_current_amd64.deb
    sudo apt-get install -f  # Fix dependencies
    ```
  </Accordion>
</AccordionGroup>

### Permission errors

If you encounter permission issues during installation:

```bash
# Fix npm permissions (don't use sudo with npm install -g)
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH

# Add to your shell profile (~/.bashrc, ~/.zshrc, etc.)
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
```

Then reinstall:

```bash
npm install -g browser-debugger-cli@alpha
```

### Port conflicts

If you see port already in use errors:

```bash
# Check what's using the default CDP port (9222)
lsof -i :9222

# Kill the process or use a different port
bdg example.com --port 9223
```
